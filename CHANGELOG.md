# Changelog

このファイルでは、Graveyard of Marksmen 無料ブラウザ版の主な変更履歴を記録します。

## [v1.52] - 2026-07-15

### Changed

- 戦意喪失の条件を変更。
- 旧条件「司令の上下左右4方向のうち3方向以上が墓標で塞がれたら敗北」を廃止。
- 新条件「司令以外の味方駒4体のうち3体以上が墓標になったら敗北」を採用。
- ルールガイド内の勝敗説明を新条件に合わせて更新。
- CPU評価用のシミュレーションにも新しい戦意喪失条件を反映。

## [v1.51] - 2026-07-13

### Fixed

- Chrome / LINE内ブラウザで縦スクロールできない問題への追加対策。
- ページ本体ではなく、アプリ全体を包む専用スクロール領域 `.appScroll` を追加。
- `html` / `body` は固定し、`.appScroll` 側で縦スクロールする構造に変更。
- 盤面・セルでは縦スクロールを通し、横パンを抑える `touch-action: pan-y` を指定。
- ボタン・セレクトはタップ操作用に `touch-action: manipulation` を指定。

## [v1.50] - 2026-07-13

### Fixed

- ChromeやLINE内ブラウザで縦スクロールできなくなる問題への対策。
- ページ全体の `touch-action` 指定を削除。
- 縦方向の `overscroll-behavior` 指定を削除。
- ページの縦スクロールをブラウザ標準の `overflow-y:auto` に戻した。
- 「盤下の方向ボタン」という古い説明文を削除。

## [v1.49] - 2026-07-13

### Fixed

- 初期向き選択中、選択中の駒本体を再タップすると現在の向きで確定する処理を明示化。
- セル全体のクリック処理だけでなく、駒本体にも直接タップ処理を追加。
- 確定時に操作履歴へ「初期向きを○○に決定」と表示。

## [v1.48] - 2026-07-13

### Changed

- 初期向き選択の矢印ボタンを、駒本体に重ねずセル内の駒周囲へ配置。
- 盤面の外には方向ボタンを追加しない仕様に変更。
- 黒い現在向き矢印バッジを削除。
- 選択中だけ駒本体を少し小さくし、方向選択ボタンとの重なりを避けるよう調整。
- 右側見切れ対策と更新・離脱時の確認処理を維持。

## [v1.47] - 2026-07-13

### Added

- ページ更新・離脱時に確認が出る処理を追加。

### Fixed

- 右側が見切れる問題を調整。
- 盤面の最大幅をスマホ画面内に収まりやすい値へ変更。
- 初期向き選択中、駒を少し小さく表示して矢印との重なりを軽減。
- 向き選択の矢印ボタンを少し外側寄りに配置。

## [v1.46] - 2026-07-13

### Fixed

- 左右に少しスクロールできる問題を抑制。
- 盤面、パネル、ガイド周辺の横はみ出しを抑制。
- スマホの横方向パンを抑制。
- 初期配置中、選択中の同じ駒を再タップすると選択解除する処理を追加。

## [v1.45] - 2026-07-13

### Changed

- 盤下の方向ボタンを削除。
- 向きを決定する盤上ボタンを少し小さく変更。
- 方向ボタンがセル外にはみ出さないように調整。
- 横方向に少しスクロールできる余白が出にくいように調整。
- 各駒に現在の向きを示す小さな矢印バッジを追加。
- 駒の正面マーカーを少し強調。

## [v1.44] - 2026-07-13

### Added

- 2人対戦のセットアップを「先手A → 後手B → 試合開始」の順に変更。
- 先手Aの配置確定後、先手Aの駒をいったん非表示にする処理を追加。
- 後手Bの配置確定後、全駒を公開して試合開始する処理を追加。
- 初期配置時の向き変更用に、盤下の方向ボタンも表示。

### Changed

- CPU戦の流れは従来どおり維持。

## [v1.43] - 2026-07-13

### Added

- 無料ブラウザ版 β として公開用ファイルを作成。
- GitHub / Cloudflare Pages で公開できる単体HTML構成に整理。
- `index.html`、`README.md`、`.gitignore` を含むアップロード用ZIPを作成。
